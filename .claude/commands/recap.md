name: recap
description: 別プロジェクトの直近セッション履歴を取り込み、文脈を共有する。チーム間の障害対応連携などに使用。
---

## When to Use

- 別チーム・別プロジェクトの直近セッション内容を把握したいとき
- 障害対応や設計議論の内容をチーム間で共有するとき
- ユーザーから「FE の履歴を見て」「別プロジェクトのセッションを確認して」等の指示を受けたとき

## Steps

1. **対象プロジェクトの確認**
   - ユーザーに対象プロジェクトのパスを確認する（未指定の場合）
   - 例: `~/projects/other-project`

2. **セッション履歴の取得**

   ```bash
   kizuna recap --project <path>
   ```

   - 量が多い場合は `--limit <n>` で末尾 N チャンクに絞る

3. **内容の要約と報告** — 以下の形式:

   ```
   ## セッション共有レポート

   **プロジェクト**: <project name>
   **セッション日時**: <timestamp>

   ### 概要
   - <セッションで議論された主要トピックの箇条書き>

   ### 重要な決定事項
   - <決まったこと、合意事項>

   ### 当プロジェクトへの影響
   - <現在のプロジェクトで対応が必要なこと>
   ```

4. **次のアクションの提案**
   - セッション内容を踏まえて、現在のプロジェクトで必要な対応を提案する

## Decision Rules

- 対象プロジェクトに `.kizuna/memory.db` が存在しない場合はその旨を報告する
- 現在のプロジェクト自身の recap（`--project` なし）も可能
- 取得した履歴は要約して報告する（生データをそのまま出力しない）
