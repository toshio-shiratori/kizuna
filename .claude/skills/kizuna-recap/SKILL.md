name: recap
description: 別プロジェクトの直近セッション履歴を取り込み、文脈を共有する（クロスプロジェクト専用）。
---

## When to Use

- 別チーム・別プロジェクトの直近セッション内容を把握したいとき
- 障害対応や設計議論の内容をチーム間で共有するとき
- ユーザーから「FE の履歴を見て」「別プロジェクトのセッションを確認して」等の指示を受けたとき

**Note**: 自プロジェクトの前セッション振り返りには `/session-start` を使用すること。

## Steps

1. **対象プロジェクトの確認**
   - ユーザーに対象プロジェクトのパスを確認する（必須）
   - 例: `~/projects/other-project`

2. **セッション履歴の取得**

   ```bash
   kizuna recap --project <path>
   ```

   **オプション:**
   - `--sessions <n>`: 直近 N セッション分を取得（デフォルト: 1）
   - `--session <id>`: 特定セッション ID を指定
   - `-n, --limit <n>`: セッションあたりの末尾 N チャンク（デフォルト: 5）
   - `--no-limit`: チャンク制限なしで全文取得

3. **内容の要約と報告** — 以下の形式:

   ```
   ## セッション共有レポート

   **プロジェクト**: <project name>
   **セッション日時**: <timestamp>

   ### 概要
   - <セッションで議論された主要トピックの箇条書き>

   ### 重要な決定事項
   - <決まったこと、合意事項>

   ### 当プロジェクトへの影響
   - <現在のプロジェクトで対応が必要なこと>
   ```

4. **次のアクションの提案**
   - セッション内容を踏まえて、現在のプロジェクトで必要な対応を提案する

## Decision Rules

- 対象プロジェクトに `.kizuna/memory.db` が存在しない場合はその旨を報告する
- `--project` パラメータは必須（自プロジェクトの振り返りには `/session-start` を使用）
- 取得した履歴は要約して報告する（生データをそのまま出力しない）
- 複数セッションを取得した場合はセッションごとに要約する
