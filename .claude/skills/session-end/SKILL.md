name: session-end
description: セッション終了時の引き継ぎ記憶の作成。未コミット変更の確認、成果要約、次回への申し送りを行う。
---

## When to Use

- セッションを終了するとき
- ユーザーから「セッション終了」「終わり」「おしまい」等の指示を受けたとき

## Steps

1. **未コミット変更の確認**

   ```bash
   git status --short
   git diff --stat
   ```

2. **未プッシュコミットの確認**

   ```bash
   git log --oneline @{upstream}..HEAD 2>/dev/null
   ```

3. **オープン PR の確認**

   ```bash
   gh pr list --state open --author @me --limit 5
   ```

4. **セッション終了レポートの報告** — 以下の形式:

   ```
   ## セッション終了レポート

   ### 今回の成果
   - <このセッションで行ったことの箇条書き>

   ### 残作業
   - 未コミット変更: あり/なし（概要）
   - 未プッシュコミット: N 件
   - オープン PR: #N（状態）

   ### 次回への申し送り
   - <次のセッションで着手すべきこと>
   - <作業途中の場合、どこまで進んだか>
   ```

5. **コミット提案**（未コミット変更がある場合）
   - Conventional Commits 形式のコミットメッセージを提案
   - コミットするかどうかユーザーに確認する

## Decision Rules

- 品質検証（tsc/test/lint/format）はこのスキルでは実行しない（`/run-tests` を使用）
- 未コミット変更を残してセッション終了する場合、その旨を明示する
- コミット・プッシュはユーザーの確認なく行わない
